## Critical Cleanup Needed

### 1. Remove or soften unsupported production claims

Avoid phrases unless proven in repo:
- production-grade
- clinically validated
- HIPAA compliant
- state-of-the-art
- guaranteed zero data egress
- 100% offline
- immutable/tamper-proof logs
- benchmark numbers without logs
- supports FP8 / TMA / production inference unless implemented and tested

Use instead:
- prototype
- reference implementation
- local-first design
- offline-capable direction
- portfolio project
- compliance-aware design pattern
- tested scaffold
- experimental benchmark path

### 2. Standardize README sections

Every top repo should include:

- What it is
- What it demonstrates
- What it does not claim
- Quick start
- Safety/privacy boundaries
- Tech stack
- Status
- Roadmap
- License
